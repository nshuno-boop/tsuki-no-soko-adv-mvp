# PHASE 5 REVIEW REPORT

Phase 5 Alpha Polish の実装後レビュー。6つの観点で静的確認し、今回は進行・推理構造を壊さない軽微修正だけを反映した。

## 概要

- 対象ブランチ: `phase5-alpha-polish`
- 対象バージョン: `v0.5-alpha`
- レビュー範囲: シナリオ、推理公平性、UX、Ren'Py技術面、アートディレクション、リリース文書
- 結論: P0の進行不能・起動不能リスクは見つからなかった。実機通しプレイ前に直す価値が高いP1は軽微な範囲で対応した。

## A. Scenario Editor Agent

### 良い点

- シロワの「空気を共有する街」というテーマが、プロローグからTrue Endingまで一貫している。
- セナは代表と母親の二重の責任が見え、単純な悪役になっていない。
- 徹の本心とノアの感情回収は、Alphaとして必要な芯がある。

### 気になる点

- 章末まとめがやや説明的に見える箇所がある。
- True Endingでセナが崩れてから結末へ向かう流れは、もう一拍あってもよい。
- Normal / Bad Endingは短く、結果説明に寄りやすい。

### 軽微修正候補

- 章末まとめを「調査まとめ」から澪のメモ調に寄せる。
- True Endingにセナが言葉を失う短い間を入れる。
- Normal / Bad Endingに、結果だけでなく人物の反応を一言加える。

### 後回しでよい改善案

- 徹の監査ファイル本文や過去ログを増やす。
- ノアとセナの終盤会話をBetaで少し厚くする。

## B. Mystery Fairness Agent

### フェアな点

- 死因、ALMAの判断材料、白兎3号の偽装、人為操作、保守モード仕様が順に積み上がる。
- Q6は複数証拠でアリバイを崩すため、単独証拠だけで断罪しない。
- ALMAは「嘘をついたAI」ではなく「偽入力を正しく処理したAI」として一貫している。

### まだ説明不足な点

- 緊急減圧の時刻が、ログ・本文・タイムライン間で揺れていた。
- 人員位置ログは、ALMAが生体反応を無視したように読める恐れがあった。
- Q7は動機確定前に犯人を問うため、橋渡しがあると納得しやすい。

### プレイヤーが誤解しそうな点

- 重要証拠表示にQ6/Q9の一部証拠が含まれず、True Ending条件と印象がずれる。
- 複数証拠推理で必要数が画面に出ず、総当たり感が出る可能性がある。

### 軽微修正候補

- 緊急減圧時刻を22:31に統一する。
- 人員位置ログと保守モード記録の説明文を1文補う。
- Q7の問題文に「動機は次に詰める」旨を加える。
- Q6/Q9に関わる証拠を重要証拠リストに追加する。

## C. UX / Player Guidance Agent

### 分かりやすい点

- 第3章ハブは目的、聞き込み進捗、おすすめ行動がまとまっている。
- 聞き込み対象6人は役職つきで並び、条件未達時の不足表示もある。
- 証拠品一覧、人物メモ、章末メモは情報整理に役立つ。

### 迷いそうな点

- 第3章の「4人以上」条件は、残り2人の意味がやや曖昧に見える。
- 「白兎3号を調べる」が実質的な章進行ボタンであることが少し弱い。
- 複数証拠問題は必要証拠数が見えない。

### 表示文言の改善候補

- 「Q6・Q8・Q9はTrue Endingに重要」というメタ表現を、作中寄りの整理文にする。
- 複数証拠画面に「選択中件数」と「必要数」を表示する。
- 第4章への進行ボタンを明示的にする。

### 軽微修正候補

- ハブの文言を少し自然化する。
- モーダル画面のz-orderを上げ、目的表示との重なりを避ける。

## D. Ren'Py Technical Agent

### 技術的に良い点

- label / jump / call screen の主要導線は揃っている。
- 証拠データは `game/evidence.rpy` に集約され、ID参照中心で保守しやすい。
- 日本語フォント設定、画像パス、開発用デバッグメニューは概ね整っている。

### 潜在バグ

- `objective_overlay` がモーダル画面上に重なる可能性があった。
- `interview_flags`、`interview_done`、`deduction_correct_ids` がsetへ直接 `.add()` されていた。
- 第3章ハブの必要聞き込み数が定数ではなく直書きだった。

### lintで拾えない可能性のある注意点

- GitHub Actionsは仮素材を生成してから整合性チェックするため、生成物の差分ずれまでは検出しない。
- `screens.rpy` 内の `Frame(...)` 参照や証拠アイコン参照の網羅検査はまだ限定的。

### 軽微修正候補

- モーダル画面に `zorder 20` を追加する。
- set更新はコピーして再代入するヘルパーへ寄せる。
- check_project_integrityにset直接変更の警告を追加する。

## E. Art Direction Agent

### 現状の良い点

- 「冷たい月面都市に残る人間の体温」という軸が文書化されている。
- ALMAは人型ではなく、波形・リング・端末UIとして扱えている。
- 素材管理表があり、placeholder / needed / final の運用に移れる。

### 本素材化前に整理すべき点

- 仮背景は色差中心で、酸素工房、外口、影井戸などの固有モチーフ差がまだ弱い。
- 仮立ち絵は色で判別できるが、輪郭差は本素材化前に強めたい。
- ALMAのidle / speaking差分は、発光や波形密度でもう少し差を出したい。

### 画像生成プロンプトで不足している点

- VN用の下部テキストボックス領域、左右立ち絵の視認性、中央焦点の指定。
- キャラ差分の一貫性、透明PNG、固定アンカー位置の指定。
- アイコン素材の個別プロンプト。
- ALMAに対する `no eyes`, `no mouth`, `no mascot`, `no humanoid avatar` の明示。

### 軽微修正候補

- 今回は本素材に触れず、プロンプト文書に不足観点を追記する。

## F. Release / Documentation Agent

### 分かりやすい点

- READMEはPhase 2からPhase 5までの経緯、フォント、静的チェック、GitHub Actionsを説明している。
- PLAYTEST_CHECKLISTは実機確認の観点として使える粒度になっている。
- FONT_LICENSEとgame/fonts/READMEでOFLフォント方針が分かる。

### 足りない説明

- ルートにある起動ショートカット `Start_TsukiNoSoko_ADV_MVP.cmd` の説明がファイル構成にない。
- Phase 5変更後の通しプレイは未確認だが、READMEの一部表現が実機確認済みに寄りすぎている。
- ART_DIRECTIONに「フォントファイルは追加せず」という古い方針が残っている。

### Alpha PRとして必要な追記

- Phase 5レビュー報告書の位置をREADMEへ明記する。
- Alpha時点の既知制限として、Phase 5後の再通しプレイ未確認を明確にする。

### 軽微修正候補

- README、PLAYTEST_CHECKLIST、ART_DIRECTION、IMAGE_PROMPTS、ASSET_MANIFESTに最小限の追記を行う。
